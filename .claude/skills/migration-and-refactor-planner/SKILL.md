# Migration & Refactor Planner

## Когда использовать

- Файл >50 КБ нужно разбить (unitymeshlab/GroupedShellTransfer.cs — 213 КБ)
- Переезд с unity 6000.0 на LTS
- Ренейм namespace (LightmapUvTool → SashaRX.MeshLab)
- Добавление Tests/Editor/ в пакет без тестов
- Извлечение тула в UPM-пакет
- Замена хрупкого кода (ручной JSON → JsonUtility)

## Шаблон плана

```
## Миграция: [название]
### Цель
[Что и зачем]
### Шаги
1. [ ] Шаг — файлы: [список]
### Риски
- [риск] → [митигация]
### Проверка
- [ ] Компиляция
- [ ] Тесты
- [ ] Ручная проверка: [сценарий]
```

## Плейбук: декомпозиция

1. Определить блоки по #region/методам/зависимостям
2. Для каждого блока: имя, файл, поддиректория
3. `internal` доступ внутри asmdef
4. По одному блоку за коммит
5. Проверка компиляции после каждого

## Плейбук: добавление тестов

1. Создать Tests/Editor/ + asmdef с defineConstraints: ["UNITY_INCLUDE_TESTS"]
2. Smoke-тесты: инстанцирование классов
3. Regression-тесты: сначала тест, потом фикс

## Плейбук: извлечение в UPM

1. Структура по unity-package-architect
2. Скопировать код, сменить namespace
3. Убрать хардкод путей, Resources.Load
4. package.json + README + CHANGELOG + LICENSE
5. Проверить git URL установку

## Плейбук: ренейм namespace

1. grep все файлы
2. Массовая замена
3. Обновить rootNamespace в asmdef
4. Один атомарный коммит

## Правила

- Каждый шаг = отдельный коммит
- Не смешивай фичи с рефакторингом
- >5 шагов — разбей на несколько PR
